## 🏗️ System Flow & Architecture

Aura Studio is built as a serverless, client-heavy application. The React frontend directly orchestrates calls to the Gemini API for multimodal generation and communicates with Firebase for authentication and data persistence.

## 🧩 Component Breakdown

### 1. Frontend (React 19 + Vite + Tailwind CSS)
- **User Interface:** Handles user input, configuration (platform selection, tone, pro-mode toggles), and displays the interleaved multimodal output.
- **AI Service Layer (`aiService.ts`):** Manages the orchestration of Gemini API calls. It handles the sequential and parallel execution of text, image, and audio generation, ensuring rate limits are respected (e.g., batching TTS requests).
- **Firebase Service Layer (`firebase.ts`):** Manages the connection to Firebase, handling authentication state, Firestore CRUD operations, and Cloud Storage uploads.

### 2. Google AI (Gemini API via `@google/genai`)
The application uses the official `@google/genai` SDK directly from the client side to interact with three distinct models:
- **`gemini-3-flash-preview`**: Acts as the "brain". It analyzes the source text, generates a summary, and outputs a strictly typed JSON schema containing the platform-specific captions, hashtags, and highly detailed visual descriptions (prompts) for the image model.
- **`gemini-2.5-flash-image`**: Takes the visual descriptions generated by the text model and produces contextual, high-quality images in Base64 format.
- **`gemini-2.5-flash-preview-tts`**: Takes the generated captions and produces natural-sounding voiceovers in raw PCM format, which the frontend converts to playable WAV files.

### 3. Backend (Firebase)
- **Firebase Authentication:** Secures the application using Google Sign-In, ensuring that users can only access and modify their own generated content.
- **Cloud Storage:** When a user saves a campaign, the Base64 images and audio files generated by Gemini are uploaded to Cloud Storage to prevent Firestore document size limits (1MB) from being exceeded.
- **Cloud Firestore:** Stores the campaign metadata, including the text captions, hashtags, summaries, and the download URLs for the media stored in Cloud Storage. Security rules ensure data isolation between users.
